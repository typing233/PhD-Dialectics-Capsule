# PhD Dialectics Capsule - 论证进化健身房

> 逼迫研究假设在AI的批判性提问与漏洞攻击中不断进化的思维健身房，帮助学者锻造严密且可发表的论证树。

## 🎯 项目简介

PhD Dialectics Capsule 是一个AI驱动的苏格拉底式对话系统，旨在帮助博士生将模糊的研究想法外化并逐步结构化。通过多轮对话动态生成可视化的假设演化树，提供一键魔鬼代言人攻击模式，内置跨学科嫁接器，以及答辩预演室，最终帮助用户产出严密、可检验的研究假设。

## ✨ 核心功能

### 1. 🎯 苏格拉底式对话
- AI驱动的多轮问答系统
- 引导用户将模糊想法逐步清晰化
- 根据对话阶段智能调整提问策略
- 自动从对话中提取论证节点

### 2. 🌳 假设演化树
- 动态生成可视化论证树
- 支持D3.js交互式可视化
- 节点类型：假设、前提、证据、反驳、修正
- 显示论证强度和脆弱度指标
- 可同步对话自动生成

### 3. 👿 魔鬼代言人模式
- 一键攻击整个论证体系
- 识别五种漏洞类型：
  - 逻辑谬误 (Logical Fallacy)
  - 证据缺口 (Evidence Gap)
  - 假设薄弱 (Assumption Weakness)
  - 方法缺陷 (Methodological Flaw)
  - 替代解释 (Alternative Explanation)
- 生成漏洞热力图
- 提供改进建议

### 4. 🎤 答辩预演室
- 基于论证树自动生成质询问题
- 覆盖五类问题：
  - 研究方法
  - 文献综述
  - 前提假设
  - 研究贡献
  - 未来工作
- 自动评估回答质量
- 提供评分、优点和改进建议

### 5. 🔬 跨学科嫁接器
- 从8个学科获取范式灵感：
  - ⚛️ 物理学 (热力学、动力学、系统论)
  - 🧬 生物学 (进化论、生态学、遗传学)
  - 💻 计算机科学 (算法、机器学习、网络理论)
  - 📊 经济学 (博弈论、激励机制、市场分析)
  - 🧠 心理学 (认知偏差、行为模式、决策科学)
  - 👥 社会学 (网络分析、社会资本、扩散理论)
  - 🔢 数学 (动力系统、混沌理论、拓扑学)
  - 📚 语言学 (话语分析、框架理论、叙事结构)
- 计算相关度评分
- 提供具体应用建议

## 📦 安装与运行

### 前置要求
- Node.js >= 18.0.0
- npm >= 9.0.0

### 安装步骤

1. **克隆项目**
```bash
cd PhD-Dialectics-Capsule
```

2. **安装后端依赖**
```bash
cd backend
npm install
```

3. **安装前端依赖**
```bash
cd ../frontend
npm install
```

1. **启动后端服务** (端口 3001)

3. **访问应用**
打开浏览器访问 http://localhost:3000

1. **构建后端**
```bash
cd backend
npm run build
npm start
```

2. **构建前端**
```bash
cd frontend
npm run build
```

## 🚀 使用指南

### 第一步：创建项目
1. 访问首页，点击"新建研究项目"
2. 输入项目名称和描述
3. 点击"创建项目"

### 第二步：苏格拉底式对话
1. 在对话界面中，系统会引导你逐步清晰化研究想法
2. 按照提示回答问题，系统会根据对话阶段调整策略：
   - **初始阶段**：澄清核心概念和研究问题
   - **深化阶段**：探索前提假设和证据基础
   - **精炼阶段**：探讨研究意义和多元视角
3. 对话会自动同步到论证树

### 第三步：查看论证树
1. 点击导航栏的"论证树"
2. 查看可视化的论证结构
3. 点击节点查看详细信息：
   - 论证强度
   - 脆弱度评分
   - 版本历史
4. 点击"同步对话"从对话中提取新节点

### 第四步：魔鬼代言人攻击
1. 点击导航栏的"魔鬼代言人"
2. 点击"一键攻击"让AI审查你的论证
3. 查看攻击结果：
   - 高/中/低风险等级
   - 问题描述
   - 改进建议
4. 查看漏洞热力图，直观了解论证弱点分布

### 第五步：跨学科灵感
1. 点击导航栏的"跨学科嫁接器"
2. 输入你的研究领域（可选）
3. 点击"探索灵感"获取其他学科的范式建议
4. 点击卡片展开详情，了解具体应用场景

### 第六步：答辩预演
1. 点击导航栏的"答辩预演"
2. 点击"生成问题"获取预演问题
3. 选择问题，输入你的回答
4. 点击"提交并评估"获取AI反馈：
   - 总体评分
   - 优点总结
   - 改进建议

## 🎯 使用示例

### 示例对话流程

**用户**：我想研究社交媒体使用与青少年抑郁之间的关系。

**AI助手**：你能否更清晰地阐述这个假设？核心变量之间的因果关系是什么？

**用户**：我的假设是每天使用社交媒体超过2小时的青少年，抑郁症状得分显著更高。可能的机制是社会比较导致的自卑感。

**AI助手**：你的假设基于哪些核心假设？这些假设是否可检验？
- 你假设社交媒体使用是因，抑郁是果，如何排除反向因果？
- 你假设社会比较是中介变量，如何测量这个机制？

**用户**：我计划使用纵向追踪设计，间隔6个月测量两次。社会比较使用专门的量表。

**AI助手**：目前有哪些实证证据支持你的假设？证据的强度如何？

**用户**：根据一些横断面研究显示存在相关，但纵向研究证据不一致。

**AI助手**：💡 提示：你的论证已经逐步成型。考虑总结一下：
1. 核心假设是什么？
2. 关键证据有哪些？
3. 潜在的弱点在哪里？

### 示例攻击结果

**节点**："社交媒体使用导致抑郁症状增加"

**攻击1**：
- 类型：逻辑谬误
- 严重度：高
- 描述：存在因果关系推断的逻辑跳跃，可能混淆相关性与因果性
- 建议：建议明确因果机制，考虑反向因果或第三方变量

**攻击2**：
- 类型：证据缺口
- 严重度：中
- 描述：关键假设缺乏直接的实证支持
- 建议：建议补充预实验数据或引用更直接的文献支持

### 示例跨学科灵感

**来源领域**：物理学
- 范式：热力学熵增原理
- 方法论：将系统视为能量耗散结构，分析其演化趋势
- 应用：可以用于分析组织衰落、信息扩散衰减等现象
- 相关度：85%

**来源领域**：社会学
- 范式：社会网络分析
- 方法论：分析关系结构、中心性、传播路径
- 应用：可以用于分析信息传播、影响力扩散、社会资本等
- 相关度：92%

## 🔧 配置说明

- 后端默认端口：3001
- 前端默认端口：3000

如需修改端口，请编辑：
- 后端：`backend/src/index.ts` 中的 `PORT` 变量

### 数据库配置
- 默认使用 SQLite 数据库
- 数据库文件存储在 `backend/data/dialectics.db`
- 首次运行时自动创建数据库和表
